    Toolchain dependencies

    These are the versions of the toolchain software I'm using to develop the app skeleton.

    NodeJS 8.9.4

    NPM 5.6.0 (included in nodejs)

    Yarn 1.3.2

    angular-cli 1.6.3 (will be installed locally by yarn/npm if not already available)

    Managing node toolchains

    I use nvm (https://github.com/creationix/nvm) on linux (it also supports mac) to manage nodejs and toolchain installations. Unfortunately it doesn't support windows, though there are a couple of alternatives listed in their readme. Looking in the related issue seems to indicate that it works fine in WSL or the other POSIXy environments for windows, but I haven't tested it.

    Source code

    The application code is in a temporary Bitbucket repository at https://bitbucket.csiro.au/users/squ022/repos/ng-portal-temp.

    It should be moved into a Bitbucket project or github repo soon.

    Develop

    After checking out the source, you need to run the following commands in the new working directory to install dependencies and run the application in dev mode.

    ?
    cd ng-portal
    yarn install
    ...
    yarn start

    Assuming it all works, this will start a development server on http://localhost:4200, which will monitor the sources for changes and automatically reload the application as required.

    You can build a production version using

    ?
    yarn build

    This creates a dist directory in the project directory that contains all of the compiled and minified javascript and other assets. Deploy the contents of the dist directory on a web server to test it. If you have python installed you can run a simple HTTP server in the dist directory like this.

    ?
    cd dist
     
    # Python3
    python -m http.server
     
    # Python2
    python -m SimpleHTTPServer

    This will not do automatic recompilation or hot reloading.
